# 참고 사이트 분석 — 오예리 특허법률사무소

- 분석 대상: https://lawer-website-gamma.vercel.app/
- 분석일: 2026-07-15
- 목적: 동일한 구조·디자인으로 재현하기 위한 페이지별 분석 문서

---

## 1. 개요

특허법률사무소(변리사) 홍보 + 상담 접수용 웹사이트. 총 5개 페이지로 구성된 정적 중심 사이트이며, 유일한 동적 기능은 상담 신청 폼(`POST /api/consult`) 하나다.

| 라우트 | 페이지 | 역할 |
|---|---|---|
| `/` | 홈 | 히어로, 실적 수치, 6대 업무분야, 진행절차, FAQ, CTA |
| `/about` | 사무소 소개 | 대표 프로필, 강점 3가지, 오시는 길 |
| `/services` | 서비스 안내 | 6대 분야 상세 (절차 + 필요서류 + CTA) |
| `/contact` | 상담 신청 | 상담 폼 + 빠른 상담 채널 안내 |
| `/privacy` | 개인정보처리방침 | 8개 조항 텍스트 페이지 |

## 2. 기술 스택

| 항목 | 내용 | 근거 |
|---|---|---|
| 프레임워크 | **Next.js (App Router) + Turbopack** | `_next/static/chunks/turbopack-*.js`, RSC payload(`__next_f`) |
| 스타일링 | **Tailwind CSS v4** | `--tw-*` 변수, `lab()` 색상 폴백, CSS 단일 번들 |
| UI 라이브러리 | 없음 (순수 Tailwind) + **lucide-react** 아이콘 | JS 청크 내 `lucide` 문자열, `strokeWidth:1.5` 패턴 |
| 폼 검증 | **Zod** (클라이언트 + 서버 양쪽) | JS 청크 내 zod 스키마, API 400 응답 |
| 폰트 | `next/font` — **Nanum Myeongjo**(Google, 제목/serif) + **Pretendard**(로컬, 본문/sans) | `--font-nanum`, `--font-pretendard` CSS 변수, 다수의 self-hosted woff2 |
| 이미지 | `next/image` (`/_next/image?url=...`) | 로고·프로필 이미지 |
| 배포 | Vercel | 도메인 |
| SEO | 페이지별 metadata, OG 이미지(1200×630), `sitemap.xml`, `robots.txt`, `icon.svg` | 각 응답 확인 |

렌더링은 전부 서버 렌더(정적)이고, 클라이언트 컴포넌트는 상담 폼·모바일 메뉴 정도로 최소화되어 있다. FAQ 아코디언은 JS 없이 네이티브 `<details>/<summary>` 6개로 구현.

## 3. 디자인 시스템

### 3.1 색상 (`:root` CSS 변수 — globals.css에 정의, Tailwind 클래스로 사용)

```css
:root {
  --background: #fff;
  --foreground: #1a2233;
  --navy-900: #0b1f3a;   /* 히어로 그라디언트 시작, 최심 배경 */
  --navy-800: #10294c;   /* CTA 배너 배경, 본문 헤딩 텍스트 */
  --navy-700: #16386a;   /* 주 버튼 배경 */
  --navy-600: #1e498a;   /* 포커스 보더, 보조 */
  --navy-50:  #eef3fa;   /* 밝은 배경 칩 */
  --gold-700: #846818;   /* 포인트 텍스트(라벨, *) */
  --gold-600: #b8912f;
  --gold-500: #c8a34e;   /* 골드 포인트(구분선 그라디언트, 강조 버튼) */
  --gold-100: #f6edd8;
  --muted:    #5b6577;   /* 보조 텍스트 */
  --line:     #e4e8f0;   /* 보더 */
  --surface:  #f7f9fc;   /* 섹션 교차 배경 */
}
```

네이비(신뢰) + 골드(전문성) 투톤. 카카오 버튼만 예외적으로 arbitrary 값 `bg-[#fee500] text-[#3c1e1e]` 사용.

### 3.2 타이포그래피

- 제목(h1~h3): `font-serif` = Nanum Myeongjo — 법률사무소 특유의 격식 있는 명조체
- 본문: Pretendard, `line-height: 1.65`, `word-break: keep-all` (한국어 줄바꿈)
- 히어로 h1: `text-3xl md:text-[2.75rem] font-bold leading-[1.4] tracking-tight`
- `html { scroll-behavior: smooth; scroll-padding-top: 88px }` — services 앵커 이동 시 헤더 높이 보정

### 3.3 레이아웃 규칙

- 콘텐츠 폭: `mx-auto max-w-6xl px-5` (좁은 텍스트 섹션은 `max-w-3xl`)
- 섹션 수직 여백: `py-20`
- 섹션 배경 교차: 흰색 ↔ `bg-surface`(+ `border-y border-line`) ↔ 네이비
- 카드: `rounded-xl border border-line bg-white` 패턴 반복

## 6. 상담 폼 & API

**필드**: 이름* / 연락처*(tel) / 이메일(선택) / 상담 분야*(select — 특허·상표·디자인·심판·소송·해외출원) / 문의 내용*(textarea) / 개인정보 동의 체크박스*(→ `/privacy` 링크 포함)

**Zod 스키마** (번들에서 복원한 실제 규칙):

```ts
z.object({
  name: z.string().trim().min(2, "이름을 2자 이상 입력해 주세요.").max(40, "이름이 너무 깁니다."),
  phone: z.string().trim().regex(/^0\d{1,2}-?\d{3,4}-?\d{4}$/, "올바른 연락처를 입력해 주세요."),
  email: z.string().trim().email("올바른 이메일을 입력해 주세요.").or(z.literal("")).optional(),
  service: z.enum(serviceIds, { message: "상담 분야를 선택해 주세요." }),
  message: z.string().trim().min(5, "문의 내용을 5자 이상 입력해 주세요.").max(2000, "문의 내용이 너무 깁니다."),
  privacy: z.literal(true, { message: "개인정보 수집에 동의해 주세요." }),
})
```

**동작 흐름**:
1. `useState`로 값/에러/상태(`idle | submitting | success | error`) 관리 (별도 폼 라이브러리 없음)
2. submit 시 클라이언트에서 `safeParse` → 실패하면 필드별 첫 에러 메시지를 인풋 아래 `text-red-600`으로 표시
3. 통과하면 `fetch("/api/consult", { method: "POST", body: JSON })`
4. **서버(Route Handler)도 동일 스키마로 재검증** — 빈 body 전송 시 `400 {"ok":false,"message":"입력값이 올바르지 않습니다."}` 확인됨
5. 성공 시 폼 대신 접수 완료 카드 표시("상담 신청이 접수되었습니다" + "24시간 이내에 담당 변리사가 연락드리겠습니다" + "다시 신청하기" 버튼)
6. 인풋 스타일: `rounded-md border bg-white px-4 py-3`, 에러 시 `border-red-400`, 포커스 시 `border-navy-600`

서버가 접수 건을 어디로 보내는지(이메일 발송 등)는 외부에서 확인 불가. privacy 페이지에 "이메일 발송(상담 알림) 위탁" 문구가 있어 **이메일 전송 서비스(예: Resend) 연동으로 추정**. 재현 시 수신 방식(이메일/슬랙/DB) 결정 필요.

## 8. 재현 시 프로젝트 구조 제안

```
app/
  layout.tsx            # 폰트 로드, Header/Footer/플로팅 액션 포함
  page.tsx              # 홈
  about/page.tsx
  services/page.tsx
  contact/page.tsx
  privacy/page.tsx
  api/consult/route.ts  # Zod 재검증 + 이메일 발송
  globals.css           # :root 색상 변수 + @theme
  sitemap.ts / robots.ts / icon.svg / opengraph-image.png
components/
  header.tsx (+ mobile-menu.tsx: "use client")
  footer.tsx
  cta-banner.tsx
  floating-actions.tsx  # 데스크톱 우하단 + 모바일 하단 바
  consult-form.tsx      # "use client", zod + useState
lib/
  services.ts           # 6대 분야 데이터 (id, 이름, 설명, 절차 5단계, 필요서류 3개)
  site.ts               # 전화번호, 주소, 이메일, 카카오 링크 등 상수
public/
  logo-mark.png, representative.jpg
```

**핵심 재현 포인트**
- 6대 분야는 데이터 배열 하나로 홈 카드 + services 상세 + 폼 select를 모두 구동 (id가 앵커·폼 값으로 공유됨)
- 연락처·주소 등은 `lib/site.ts` 상수로 (현재 사이트도 02-1234-5678, contact@example.com 등 전부 플레이스홀더)
- FAQ는 `<details>`로 JS 없이 구현
- 지도 영역은 원본도 플레이스홀더 — 실 지도 연동 여부는 발주자 확인 필요
